## Understanding the Fragment Workflow

### How Lesson Files Are Created

Each lesson is created as **2-3 HTML fragment files minimum** (parts a, b, c) that are later joined together by the user into a single complete HTML file. **Only create additional parts (d, e, etc.) if the lesson content absolutely requires it.** Prioritize efficiency—most lessons should be completable in 2-3 parts.

#### Fragment Structure:

**Part A (Opening):**
```html
<!DOCTYPE html>
<html lang="en">
<head>
    <!-- Complete head section: meta tags, title, CSS links, scripts -->
</head>
<body>
    <!-- Navigation bar -->
    <!-- Breadcrumb navigation -->
    <!-- Main content container opens -->
    <!-- First 3-4 major content sections -->
```

**Parts B & C (Middle Content):**
```html
    <!-- Continue content sections -->
    <!-- No HTML structure tags -->
    <!-- Just section content -->
     <!-- Maybe less parts, maybe more -->
```

**Part D (Closing):**
```html
    <!-- Final content sections -->
    <!-- Lesson navigation buttons -->
    <!-- Completion tracker -->
    <!-- Footer -->
    <!-- All JavaScript -->
     <!--this could potentially be Part E or Part B depending on amount of content -->
    </main>
</body>
</html>
```

### Why Fragments?

- **Large file management:** Lessons are 30,000+ words; fragments are easier to work with
- **Collaboration:** Multiple people could work on different parts
- **Version control:** Easier to track changes in smaller files
- **AI token limits:** Fits better within AI response limitations
- **Manual joining:** User joins fragments when ready, not automatic

### Joining Fragments

To create the complete lesson file:
1. Concatenate parts a + b + c + d in order
2. Save as single file: `lesson_name.html`
3. No editing needed - fragments are designed to fit perfectly together
4. Test the complete file in browser
