Texture Map Types in Modern Games

PBR Material Maps

Base Color / Albedo

  • Purpose: Raw surface color without lighting information
  • Resolution: Typically highest (2K-4K for heroes)
  • Format: RGB, sRGB color space
  • Critical Rule: NO lighting baked in! Pure surface color only

Normal Map

  • Purpose: Surface detail and bump information
  • Resolution: Same as albedo or half-res
  • Format: RGB, Linear color space
  • Channels: R=X, G=Y, B=Z (tangent space normals)

Roughness / Smoothness

  • Purpose: How rough/smooth surface is (controls specular spread)
  • Resolution: Can be half-res of albedo
  • Format: Grayscale, Linear
  • Values: 0=mirror smooth, 1=completely rough

Metallic

  • Purpose: Defines if surface is metal or dielectric
  • Resolution: Often lower res (512-1024)
  • Format: Grayscale, Linear
  • Values: Usually 0 or 1 (binary), rarely in-between

Ambient Occlusion (AO)

  • Purpose: Shadows in crevices and contact points
  • Resolution: Can be lower res
  • Format: Grayscale
  • Note: Often baked from high-poly, multiplied with albedo

Emissive

  • Purpose: Self-illuminated areas (screens, lights, magic)
  • Resolution: Lower res usually fine
  • Format: RGB (can be very compressed)
  • HDR: Can exceed 1.0 for bloom effects

Workflow Patterns

🎨 Modern Game Art Workflows

Workflow 1: High-to-Low Baking (AAA Standard)

1. Sculpt high-poly model (millions of polygons)
   └── ZBrush, Blender, Mudbox
   
2. Create low-poly game model (thousands of polygons)
   └── Maya, Blender, 3ds Max
   
3. UV unwrap low-poly
   └── Optimize for minimal stretching
   
4. Bake high-poly detail to maps
   └── Normal, AO, Curvature → Substance Painter, Marmoset
   
5. Hand-paint textures in Paintstorm
   └── Use baked maps as base
   └── Add artistic detail, variation, story
   
6. Export texture set
   └── Combine maps for engine
   
7. Set up material in engine
   └── Unity, Unreal, Custom engine

Workflow 2: Pure 2D Assets (UI, Sprites, Concepts)

1. Concept sketch
   └── Paintstorm or Photoshop
   
2. Refine linework
   └── Clean, game-ready lines
   
3. Color and shade
   └── Consider lighting in-game
   
4. Export at multiple resolutions
   └── @1x, @2x, @3x for different screen densities
   
5. Create sprite sheets if animated
   └── Optimize atlas packing
   
6. Import to engine with proper settings
   └── Compression, filtering, mipmaps

Workflow 3: Tiling Textures (Environments)

1. Create seamless base pattern
   └── Paintstorm with offset preview
   
2. Add variation layers
   └── Macro detail, micro detail
   
3. Generate material maps
   └── Normal from height, roughness variation
   
4. Test tiling at scale
   └── 10x10 tiles minimum
   
5. Create trim sheets if needed
   └── Modular pieces for variety
   
6. Export and implement
   └── Test in actual game environment

PBR Texture Painting ⚡

Physically Based Rendering (PBR) revolutionized game art. Instead of "painting lighting," we paint surface properties that react realistically to any lighting. Master PBR, and your assets look incredible in any environment!

Understanding PBR Principles

The Core Concept

Traditional: "This area is bright because there's a light source."
PBR: "This surface is rough metal, so it will react to light this way."

PBR separates what the surface IS (material properties) from how it's LIT (lighting calculations). This means:

  • Assets look correct in any lighting scenario
  • No need to repaint for different environments
  • Physically accurate reflections and light interaction
  • Easier for technical artists to implement

⚠️ Critical PBR Rules:
  • Never go below RGB(50,50,50) for base color (too dark)
  • Never go above RGB(240,240,240) for dielectrics (too bright)
  • Metals should have colored base color, no albedo darkening
  • Dielectrics (non-metals) should have very low metallic (0.0-0.04)
  • Use reference! When in doubt, check real-world material values

Painting PBR Textures in Paintstorm

🎨 Workflow: Creating a PBR Material Set

STEP 1: Set Up Your Canvas
─────────────────────────────
• Create new document at target resolution (2048×2048 common)
• Set color mode to RGB, 8-bit or 16-bit
• Import UV layout as reference layer
• Import baked maps if available (AO, normal, curvature)

STEP 2: Base Color (Albedo) Pass
─────────────────────────────
• Block in large material zones
  └── Use flat colors based on PBR values
  └── No lighting information!
  
• Add color variation
  └── Slight hue shifts (±10 degrees)
  └── Saturation variation in organic materials
  └── Consider surface contamination (dirt, dust)
  
• Add macro detail
  └── Large scratches, wear patterns
  └── Material transitions
  └── Decals, logos, graphics
  
• Add micro detail
  └── Small scratches, pitting
  └── Subtle noise for realism
  └── DON'T overdo it - less is more!

STEP 3: Roughness Map Creation
─────────────────────────────
• Start with base material roughness value
  └── Flat fill based on material type
  
• Add variation based on wear
  └── Worn areas = smoother (darker in roughness map)
  └── Dirty areas = rougher (lighter in roughness map)
  └── Wet areas = much smoother
  
• Detail work
  └── Scratches are usually smoother
  └── Edges get worn = smoother
  └── Cavities collect dirt = rougher
  
• Use curvature map as guide
  └── Exposed edges = wear = smoother
  └── Crevices = protection = original roughness

STEP 4: Metallic Map (if needed)
─────────────────────────────
• Usually binary: 0 for dielectric, 1 for metal
• Paint solid values based on material
• Sharp transitions OK - metals don't gradually appear
• Consider only at material boundaries
• Rare: 0.04 for specular adjustment on dielectrics

STEP 5: Normal Map Enhancement
─────────────────────────────
• If you baked from high-poly: Start there
• If creating from scratch: Use height-to-normal tools
  
• Paint height information in grayscale
  └── White = raised, Black = recessed
  
• Add details:
  └── Panel lines, rivets, bolts
  └── Fabric weave, leather grain
  └── Fine scratches (perpendicular to surface)
  └── Damage, dents, impacts
  
• Convert to normal map
  └── Use Photoshop filter or external tool
  └── Adjust strength as needed

STEP 6: Ambient Occlusion
─────────────────────────────
• If baked: Use as base
• If not: Paint contact shadows manually
  
• Darken:
  └── Crevices and gaps
  └── Where objects touch
  └── Panel intersections
  └── Deep wrinkles/folds
  
• Keep it subtle - strong AO can look baked-in

STEP 7: Emissive (Optional)
─────────────────────────────
• Only for self-illuminating areas
• Can use full RGB color
• Can exceed 1.0 for bloom effect
• Common uses:
  └── Screens, displays
  └── Lights, LEDs
  └── Magical effects
  └── Energy fields

STEP 8: Quality Check
─────────────────────────────
• View each map individually
• Check in 3D viewer with different lighting
• Ensure no baked lighting in base color
• Verify PBR value ranges
• Test with rough and smooth lighting
• Export and test in actual game engine

Common PBR Mistakes to Avoid

❌ Don't Do This!

Mistake 1: Baked Lighting in Albedo

  • ❌ BAD: Painting shadows and highlights in base color
  • ✅ GOOD: Pure surface color only, let engine handle lighting
  • Why: Baked lighting breaks under different light conditions

Mistake 2: Unrealistic Base Color Values

  • ❌ BAD: RGB(0,0,0) black or RGB(255,255,255) white dielectrics
  • ✅ GOOD: Stay in RGB(50-240) range for realistic materials
  • Why: Real materials don't reflect 0% or 100% light

Mistake 3: Wrong Metallic Values

  • ❌ BAD: 0.5 metallic "to make it shiny"
  • ✅ GOOD: 0.0 or 1.0 only (materials are metal or not)
  • Why: There's no such thing as "half metal" in nature

Mistake 4: Confusing Roughness with Brightness

  • ❌ BAD: Using roughness to make materials look darker
  • ✅ GOOD: Roughness only controls specular spread, not brightness
  • Why: Brightness comes from base color, not roughness

Mistake 5: Overdetailing

  • ❌ BAD: Every pixel has unique detail, pure noise
  • ✅ GOOD: Strategic detail with clean areas for visual rest
  • Why: Too much detail = visual mud, hard to read

Mistake 6: Inconsistent Wear Patterns

  • ❌ BAD: Random scratches everywhere with no logic
  • ✅ GOOD: Wear appears where it would naturally occur
  • Why: Believable wear tells a story

Mistake 7: Matching Roughness to Base Color

  • ❌ BAD: Dark areas in base color = dark in roughness
  • ✅ GOOD: Roughness is independent of color
  • Why: Black rubber is rough, black plastic is smooth - color ≠ roughness

Advanced PBR Techniques

Technique 1: Wear and Tear Pattern Logic

WHERE Wear Appears:
────────────────────
✓ High-traffic areas (handles, edges, corners)
✓ Areas that get gripped or touched
✓ Exposed edges that catch on things
✓ Areas subject to friction or impact
✗ Protected crevices and gaps
✗ Rarely-touched areas
✗ Newly added components

HOW to Paint Wear:
────────────────────
Base Color Layer:
  • Slightly lighter color (metal showing through)
  • Or dirt/grime buildup in protected areas
  • Edge highlights where paint chips

Roughness Layer:
  • SMOOTHER (darker value) on worn areas
  • Polished by friction
  • Especially on metal edges

Normal Map:
  • Subtle rounding of sharp edges
  • Small dents and impacts
  • Directional scratches

Example: Worn Leather Boot
────────────────────
Base Color:
  • Fresh leather: RGB(110, 70, 45)
  • Worn areas: Slightly lighter, desaturated
  • Creases: Darker accumulation
  
Roughness:
  • New leather: 0.65
  • Worn smooth areas: 0.45
  • Scuffed areas: 0.75
  
Normal:
  • Deep creases in bend areas
  • Scratches parallel to use direction
  • Scuff marks with height variation

Technique 2: Material Blending Zones

When Different Materials Meet:
────────────────────
Example: Metal plate on leather

Transition Zone (2-5 pixels):
  Base Color:
    • Metal: Proper metal color
    • Leather: Proper leather color
    • Edge: Slight color bleed (oxidation on metal, staining on leather)
  
  Roughness:
    • Sharp transition OK
    • Maybe slight smoothing at contact point (friction)
  
  Metallic:
    • HARD transition - either metal or not
    • No gradient between materials
  
  Normal:
    • Small gap/crack at interface
    • Contact wear on both materials
  
  AO:
    • Darken the crevice/contact area
    • Simulate light occlusion

Technique 3: Tri-Planar Detail Projection

For organic or complex models without clean UVs:

1. Paint detail maps in world space
   └── Top-down, side, front views
   
2. Use tri-planar projection in shader
   └── Blend based on surface normal
   
3. Benefits:
   ✓ No UV stretching artifacts
   ✓ Consistent detail density
   ✓ Easy to add/modify
   
4. Drawbacks:
   ✗ Slightly more expensive to render
   ✗ Less precise control
   
Best For:
  • Rocks, cliffs, terrain
  • Organic characters (when combined with UV)
  • Background props

Technique 4: Color ID Masking

Workflow for Complex Assets:

1. Create Color ID Map
   └── Flat colors per material zone
   └── R=255 for metal parts
   └── G=255 for leather parts
   └── B=255 for fabric parts
   └── etc.

2. In Paintstorm, use Color Range selection
   └── Select specific ID color
   └── Paint details only in that zone
   └── Ensures clean material separation

3. Benefits:
   ✓ Easy to isolate areas
   ✓ Quick selection of material zones
   ✓ No edge bleeding between materials
   ✓ Can automate with actions/scripts

4. Export final maps without ID layer
   └── ID map is workflow tool only
🎨 Pro Workflow: Always work from large to small detail. Block in major material zones, add medium-scale variation, then add fine details last. This prevents you from wasting time on details that might get covered or changed!

Skin Texturing Techniques

🎨 Painting Realistic Skin

Step 1: Base Skin Tone

Color Values for Different Skin Tones:
────────────────────────────────────
Fair Caucasian:   RGB(240, 210, 195) + variations
Medium Caucasian: RGB(232, 190, 172)
Olive/Tan:        RGB(205, 164, 133)
Light Brown:      RGB(165, 126, 100)
Medium Brown:     RGB(130, 92, 67)
Dark Brown:       RGB(90, 60, 45)
Very Dark:        RGB(55, 40, 30)

IMPORTANT: These are starting points!
• Add color variation (±15 degrees hue)
• Adjust saturation per area
• Never use pure grayscale for "white" skin
• All skin has color - reds, yellows, blues

Step 2: Subsurface Scattering Zones

Areas with More Blood Flow (redder):
✓ Nose tip
✓ Cheeks
✓ Ears (very red!)
✓ Lips (much redder)
✓ Fingertips
✓ Elbows, knees (if visible)

Areas with Less Blood (more yellow/olive):
✓ Forehead
✓ Chin
✓ Neck
✓ Chest

Areas with Blue Undertones:
✓ Around eyes (thin skin, veins visible)
✓ Temples
✓ Under cheekbones (sometimes)

Technique:
1. Block in base skin tone
2. Add subtle red layer on blood-rich areas (5-15% opacity)
3. Add subtle blue/green on areas with thin skin
4. Blend smoothly - no harsh transitions
5. More contrast = more vibrant and alive
6. Less contrast = more pale/sickly/dead

Step 3: Skin Detail Layers

Layer Order (Bottom to Top):
────────────────────────────
1. Base Skin Tone (flat color)
2. Subsurface Color Variation (red/yellow/blue zones)
3. Large Pores/Texture (10-20% opacity, subtle)
4. Fine Pores (5-10% opacity, very subtle)
5. Wrinkles and Creases (painted with care)
6. Freckles/Beauty Marks (if applicable)
7. Scars and Blemishes
8. Facial Hair/Stubble (if applicable)
9. Makeup (if applicable)
10. Dirt/Grime (environment-dependent)

Painting Each Layer:
────────────────────
Large Pores:
  • Use textured brush, 10-30 pixel size
  • Very low opacity (10-20%)
  • Slightly darker than base skin
  • More visible on nose, cheeks, forehead
  • Less visible on smooth areas (under eyes)

Wrinkles:
  • Paint in Base Color: Subtle darkening in crease
  • Paint in Normal Map: Define depth
  • Paint in Roughness: Creases are slightly rougher
  • Direction matters: Follow muscle structure
  • Depth varies: Smile lines deeper than crow's feet

Scars:
  • Base Color: Lighter, less pigmented (or darker if fresh)

Tileable Texture Creation

🔄 Making Perfect Seamless Textures

Method 1: Offset Method (Most Common)

In Paintstorm Studio:

1. Create Your Texture
   └── Paint naturally, don't worry about seams yet
   └── Keep detail in center
   └── Leave edges relatively clean

2. Apply Offset
   └── Filter > Other > Offset
   └── X: 50% of width
   └── Y: 50% of height
   └── This moves the seams to the center

3. Fix the Seam
   └── Use Clone Stamp tool
   └── Paint over visible seam
   └── Match texture patterns
   └── Blend carefully

4. Repeat if Needed
   └── Offset again to check other seam
   └── Fix any remaining issues

5. Test Tiling
   └── Create large canvas (4x4 tiles)
   └── Fill with pattern
   └── Look for repetition artifacts
   └── No obvious grid or crosses = success!

Method 2: Edge Blending

For organic textures (ground, rock, etc.):

1. Paint texture normally

2. Select edge regions (50-100 pixels)
   └── Top edge
   └── Copy and flip vertically
   └── Paste at bottom edge
   └── Blend at 50% opacity

3. Repeat for sides
   └── Left edge
   └── Copy and flip horizontally
   └── Paste at right edge
   └── Blend at 50% opacity

4. Fix corners
   └── Clone stamp from nearby areas
   └── Blend carefully

5. Add variation across entire tile
   └── Prevents obvious repeating pattern

Method 3: Procedural Noise Base

For natural materials (dirt, stone, etc.):

1. Start with tileable noise
   └── Generate Perlin/Simplex noise
   └── These are naturally tileable
   └── Multiple octaves for detail

2. Add color variation
   └── Color layers with blend modes
   └── Hue/saturation adjustments
   └── Keep subtle

3. Paint details on top
   └── Cracks, features
   └── These can cross seams (will still tile)
   └── Use offset method to check

4. Add final touches
   └── Highlights, shadows
   └── Wear patterns
   └── Localized color variation

Result: Tileable base that looks organic!

Avoiding Repetition Artifacts

Common Tiling Problems:
────────────────────────────

Problem 1: The "Cross" Pattern
Symptom: X-shaped pattern appears when tiled
Cause: Too much contrast at seams
Fix: Reduce contrast near edges, add variation

Problem 2: Obvious Grid
Symptom: Can see individual tile boundaries
Cause: Features too centered or regular
Fix: Break up symmetry, add random elements

Problem 3: Directional Flow
Symptom: Clear diagonal or vertical lines
Cause: Brush strokes too uniform
Fix: Vary brush direction, add perpendicular detail

Problem 4: High-Contrast Feature
Symptom: One bright/dark spot draws eye
Cause: Feature too unique, repeated obviously
Fix: Duplicate and vary the feature elsewhere

Solutions:
────────────────────────────
✓ Add macro variation (large-scale noise)
✓ Add micro variation (small-scale detail)
✓ Break symmetry
✓ Use multiple textures blended
✓ Add vertex color variation
✓ Use decals over tileable base
✓ Rotate/flip textures randomly in engine

Texture Blending Systems

🎨 Multi-Material Blending

Technique 1: Vertex Painting

Blend multiple tileable textures using vertex colors:

Setup:
• 4 tileable textures (RGBA channels)
  R = Grass
  G = Dirt  
  B = Stone
  A = Sand (or use Alpha for blending)

In Engine Material:
• Sample all 4 textures
• Multiply each by vertex color channel
• Add results together

Advantages:
✓ Runtime control (artists can paint in engine)
✓ Smooth transitions
✓ Infinite detail (textures tile)
✓ Very memory efficient

Disadvantages:
✗ Need sufficient vertex density
✗ Can't have hard transitions
✗ Limited to 4 materials per shader

Best For:
• Terrain
• Large outdoor environments
• Organic transitions

Technique 2: Mask-Based Blending

Use texture masks to control blending:

Setup:
• Base tileable textures (2-8 materials)
• Blend mask texture (RGBA = 4 masks)
  R channel = Where Material A shows
  G channel = Where Material B shows
  B channel = Where Material C shows
  A channel = Where Material D shows

Advantages:
✓ Pixel-perfect control
✓ Can have sharp transitions
✓ More than 4 materials possible (multi-layer)
✓ Can be painted in Paintstorm

Disadvantages:
✗ Requires UV space for mask
✗ Additional texture memory
✗ Less runtime control

Best For:
• Hero props with multiple materials
• Precise material placement needed
• Stylized looks with sharp transitions

Technique 3: Height-Based Blending

Blend based on height/normal information:

Material Shader Logic:
1. Sample height from both materials
2. Compare heights at blend boundary
3. Higher material "wins" at boundary
4. Creates natural stacking effect

Example: Snow on Rock
• Rock height: 0.5 (base)
• Snow height: 0.8 (accumulates on top)
• At blend zone: Snow appears on top of rock peaks
• Result: Natural accumulation pattern

Code Concept:
height_blend = height_A - height_B + blend_factor
blend_mask = saturate(height_blend / sharpness)
final_color = lerp(material_B, material_A, blend_mask)

Advantages:
✓ Physically realistic
✓ Natural accumulation patterns
✓ Works great for snow, moss, rust
✓ Automatic detail in transitions

Disadvantages:
✗ More expensive computationally
✗ Requires height maps for materials
✗ Can be hard to art-direct precisely

Best For:
• Natural accumulation (snow, dust, moss)
• Weathering effects
• Realistic material layering

Performance Optimization for Tileables

Compression and Memory

Texture Compression Guidelines:
────────────────────────────────

Base Color (Albedo):
Format: BC1/DXT1 (no alpha) or BC3/DXT5 (with alpha)
Quality: Can use moderate compression
Note: sRGB color space

Normal Maps:
Format: BC5/3Dc (two channel)
Quality: Higher quality needed
Note: Store only XY, reconstruct Z
      Linear color space

Roughness/Metallic/AO (Packed):
Format: BC1/DXT1 or BC7 for better quality
Quality: Roughness needs good quality
      Metallic can be more compressed
      AO can be heavily compressed
Note: Pack in RGB channels
      R = AO, G = Roughness, B = Metallic

Memory Calculation:
────────────────────────────────
1024×1024 texture memory:
• Uncompressed: 4MB (RGBA8)
• BC1: 0.5MB (8:1 compression)
• BC5: 1MB (4:1 compression)
• BC7: 1MB (4:1 compression, better quality)

Tileable Texture Strategy:
• Use higher resolution (more detail)
• Use good compression (less memory)
• Result: Better quality than unique lower-res textures!

Example Budget:
10 tileable textures at 1024×1024 BC1:
= 5MB total texture memory
Can cover entire level!

vs.

100 unique textures at 512×512 uncompressed:
= 100MB total texture memory
Only covers small area!

Fire Effect

Approach: Additive blending, animate from bottom

Base Color:

• Bottom: Narrower, denser

1-4: Ignition (small, building)

Approach: Alpha blended, animate upward

Base Color:
• Dark smoke: RGB(40, 40, 40) light smoke: RGB(200, 200, 200)
• Slight color tint OK (brown, blue)
• Subtle color variation within plume

Shape:
• Bottom: Denser, more defined
• Rising: Expands, diffuses
• Top: Very soft, dissipating
• Turbulent edges (use noise)

Alpha Channel:
• Bottom: 60-80% opacity
• Middle: 40-60% opacity

Animation Frames:
1-4: Dense, small
5-8: Expanding, thinning
9-12: Large, diffuse
13-16: Dissipating, breaking apart

Magic/Energy Effect

Approach: Additive, saturated colors, glow

Base Color:
• Core: Super bright (RGB: 255, 255, 255)
• Primary: Saturated hue (RGB: 100, 200, 255 for blue)
• Secondary: Complementary hints
• Edges: Color fade to transparency

HDR Values:
• Center can exceed 255 for bloom effect
• In Paintstorm: Paint white, will be multiplied in engine
• Creates "impossibly bright" magical look

Shape:
• Geometric or organic based on type
• Sharp edges = arcane/technical magic
• Soft edges = natural/wild magic
• Swirls and tendrils common

Alpha Channel:
• High opacity at core

Animation:
• Pulse (scale variation)
• Shimmer (opacity flicker)
• Rotation (spiral patterns)
• Flow (moving tendrils)

Impact/Explosion Effect

Approach: Multiple layers, quick animation

Initial Flash (Frame 1-2):
• Pure white or bright color
• Large, round, soft
• Very high opacity
• Quick (1-2 frames only)

Shockwave (Frame 2-5):
• Ring expanding outward
• Starts opaque, fades fast
• Color based on effect type

Debris (Frame 3-10):
• Particles flying outward

Smoke/Dust (Frame 5-16):

• Dissipating

All on separate particle systems

VFX Technical Considerations

⚡ Performance & Best Practices

Overdraw is the Enemy

Overdraw = How many times a pixel is drawn

Problem:
• 10 particles overlap at one pixel
• GPU draws that pixel 10 times
• 9 are wasted (covered by later ones)

Solutions:
✓ Reduce particle count
✓ Smaller particles where possible
✓ Don't overlap unnecessarily  
✓ Use LOD (fewer particles at distance)
✓ Clip/cull particles behind camera
✓ Use additive where appropriate (cheaper)

Alpha Blending Costs:
Cheapest → Most Expensive
1. Opaque (no alpha)
2. Alpha Test (cutout, binary)
3. Additive (blend, but doesn't need sorting)
4. Alpha Blend (requires sorting, expensive)

Beautiful art that doesn't run at 60 FPS is useless in games. Professional game artists are technical artists - they understand performance as deeply as aesthetics. Let's master the optimization strategies that separate hobbyists from professionals!

LOD (Level of Detail) Strategy

Understanding LOD Systems

LOD Philosophy:
────────────────────────────
Only render detail the player can actually see!

LOD Levels Standard:
────────────────────────────
LOD0 (Highest Detail):
  Distance: 0-5 meters
  Triangles: 100% of budget
  Textures: Full resolution
  Materials: All details active
  Use: Player can inspect closely

LOD1 (High Detail):
  Distance: 5-15 meters
  Triangles: 40-60% of LOD0
  Textures: Full or half resolution
  Materials: All details, maybe simplified
  Use: Nearby, visible but not inspected

LOD2 (Medium Detail):
  Distance: 15-40 meters
  Triangles: 20-40% of LOD0
  Textures: Half resolution
  Materials: Simplified, fewer maps
  Use: Background objects, still clearly visible

LOD3 (Low Detail):
  Distance: 40-100 meters
  Triangles: 5-15% of LOD0
  Textures: Quarter resolution
  Materials: Basic, single texture often
  Use: Far background, silhouette visible

LOD4+ (Imposters/Billboards):
  Distance: 100+ meters
  Triangles: 2 (single quad)
  Textures: Baked image of object
  Materials: Simple textured quad
  Use: Very far, just needs to exist

Example: Tree Asset
────────────────────────────
LOD0: 15,000 triangles, 2048×2048 textures
  Individual leaves, branches, bark detail
  
LOD1: 6,000 triangles, 2048×2048 textures
  Leaf clusters, simplified branches
  
LOD2: 2,000 triangles, 1024×1024 textures
  Simple crown, basic trunk
  
LOD3: 500 triangles, 512×512 textures
  Cone shape, flat leaf cards
  
LOD4: 2 triangles, 256×256 texture
  Flat billboard with tree image

Texture Optimization Strategies

🎨 Maximizing Quality, Minimizing Memory

Strategy 1: Texture Atlasing

Combine Multiple Textures:
────────────────────────────

Instead of:
❌ 10 textures × 512×512 = 10 draw calls + 10MB
   Each is its own material

Do:
✅ 1 atlas × 2048×2048 = 1 draw call + 4MB (compressed)
   All share one material
   
Savings:
• 90% fewer draw calls
• 60% less memory
• Faster rendering

When to Atlas:
✓ Small props that appear together
✓ UI elements
✓ Character accessories
✓ Particle effects
✓ VFX elements

When NOT to Atlas:
✗ Need mipmaps per texture
✗ Different tiling rates needed
✗ Different material properties
✗ Very large individual textures

Strategy 2: Channel Packing

Pack Multiple Maps into One Texture:
────────────────────────────────────

Standard Packing: ORM Map
R channel = Ambient Occlusion (or nothing)
G channel = Roughness
B channel = Metallic
A channel = (unused or other mask)

Benefits:
• 3 textures → 1 texture
• 66% memory savings
• Fewer texture samples in shader
• Industry standard practice

Alternative Packings:
────────────────────────────
RMA Pack:
R = Roughness
G = Metallic  
B = AO
A = Emissive mask (or height)

MRAE Pack (if needed):
R = Metallic
G = Roughness
B = AO
A = Emissive

Rules:
✓ Pack maps with similar importance
✓ Don't pack maps needing different compression
✓ Normal maps stay separate (need BC5 compression)
✓ Base color stays separate (needs sRGB color space)

Strategy 3: Intelligent Resolution

Not Everything Needs 4K!
────────────────────────────

Resolution Guidelines:
Hero Character Face: 4096×4096 (close-up cutscenes)
Hero Character Body: 2048×2048 (visible but not as close)
Hero Character Hands: 1024×1024 (sometimes close in first-person)
Hero Props/Weapons: 2048×2048 (inspected in inventory)

Common Enemy: 2048×2048 (body + head combined)
Common Props: 1024×1024 or shared atlas
Background Props: 512×512 or shared atlas

Environment Tileables: 1024×1024 or 2048×2048
  (High res OK because they tile infinitely)

UI Elements: 512×512 or less (screen-space, not world-space)
Icons: 256×256 or 128×128

VFX: 512×512 to 1024×1024 (usually far and fast-moving)

Calculation Example:
────────────────────────────
Bad Allocation:
Background crate: 2048×2048 = 16MB uncompressed
Player weapon: 512×512 = 1MB uncompressed
Result: Wasted memory on thing player rarely sees

Good Allocation:
Background crate: 512×512 = 1MB uncompressed  
Player weapon: 2048×2048 = 16MB uncompressed
Result: Detail where player actually looks!

Strategy 4: Compression Settings

Choose the Right Compression:
────────────────────────────

BC1/DXT1 (6:1 compression):
✓ Base color without alpha
✓ Roughness/metallic/AO packed
✗ Visible artifacts in gradients
Use: Most diffuse textures

BC3/DXT5 (4:1 compression):
✓ Base color with alpha
✓ Masked textures (foliage)
✗ Artifacts in alpha gradients
Use: Textures needing transparency

BC5/3Dc (4:1 compression):
✓ Normal maps (two-channel)
✓ Best quality for normals
✓ Reconstruct Z in shader
Use: ONLY for normal maps

BC7 (4:1 compression):
✓ High quality color
✓ High quality alpha
✓ Best general-purpose
✗ Slower to compress
✗ Not supported everywhere
Use: Hero assets on modern platforms

Mobile:
• Use ASTC compression (better quality than DXT)
• Adjust block size based on importance
• 4×4 blocks = higher quality
• 8×8 blocks = lower quality, smaller size

Strategy 5: Mipmap Optimization

Mipmaps = Pre-filtered smaller versions
────────────────────────────────────

Without Mipmaps:
❌ Texture shimmering at distance
❌ Performance hit (sampling too many texels)
❌ Moiré patterns

With Mipmaps:
✅ Smooth appearance at all distances
✅ Better performance (cache friendly)
✅ +33% memory cost (worth it!)

Mipmap Chain:
2048×2048 → 1024×1024 → 512×512 → 256×256 
  → 128×128 → 64×64 → 32×32 → 16×16 
  → 8×8 → 4×4 → 2×2 → 1×1

GPU automatically picks appropriate level based on distance

Optimization:
• Last few mips (16×16 and below): Replace with solid color
• Tint the mips to verify they're being used (debugging)
• Consider sharpening mips for better distance clarity

Draw Call Optimization

Reducing Draw Calls

Draw Calls = Major Performance Cost
────────────────────────────────────

What is a Draw Call?
CPU tells GPU: "Draw this object with this material"

Cost Breakdown:
• CPU overhead: Preparing command
• State changes: Switching textures/shaders
• GPU overhead: Starting new draw

Target Draw Calls (per frame):
Mobile: 50-100
Console: 500-1000  
PC: 1000-2000+

Optimization Strategies:
────────────────────────────────────

1. Texture Atlasing
   • Multiple objects, one texture, one material
   • 100 objects with individual textures = 100 draw calls
   • 100 objects with shared atlas = 1 draw call

2. Material Sharing
   • Different objects can share same material
   • Vary through vertex colors, secondary UVs
   • All "stone" props use one stone material

3. Static Batching
   • Combine static meshes at build time
   • 50 rocks → 1 combined mesh
   • Single draw call for entire group

4. Dynamic Batching (real-time)
   • Engine combines similar objects per frame
   • Works for small objects (< 300 verts usually)
   • Must share material

5. Instancing
   • Draw same mesh multiple times efficiently
   • One draw call for 1000 trees
   • Each can have different position/rotation
   • Requires instancing support in engine

6. LOD Reduces Active Objects
   • Distant objects swap to simpler versions
   • Or cull entirely beyond distance
   • Fewer active objects = fewer draw calls

  • Triangle count: 20k-40k (LOD0)

  • Maps required: Base Color, Normal, ORM (AO+Roughness+Metallic)
  • LODs: At least 3 levels (LOD0, LOD1, LOD2)
  • PBR values must be accurate

Must Include:

  • Character tells a story through texture
  • Realistic wear and tear
  • Proper skin texturing (if visible)
  • Fabric/material variety
  • Clean topology for animation
Component 2: Environment Props (3-5 assets)

Technical Specs:

  • Triangle count: 500-3k each (LOD0)

Must Include:

  • Modular design (combinable)
  • Consistent style and scale
  • Proper PBR materials
  • Optimized UVs (no wasted space)
Component 3: Tileable Textures (2 sets)

Technical Specs:

  • Resolution: 1024×1024 or 2048×2048
  • Maps: Base Color, Normal, ORM
  • Must tile seamlessly
  • Test with 10×10 grid minimum

Must Include:

  • Appropriate for theme
  • No visible repetition artifacts
  • Proper texel density
  • Work together cohesively
Component 4: VFX Set

Technical Specs:

  • At least 2 different effects
  • Sprite sheet: 1024×1024 (4×4 or 8×8)
  • Atlas format acceptable
  • Proper alpha channels

Must Include:

  • Animated effects (looping or one-shot)
  • Appropriate for theme
  • Optimized for performance
  • Multiple variations for variety
